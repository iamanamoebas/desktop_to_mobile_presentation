Separate Code Base

Advantages

Keeps each code base simpler

desktop doesn't have to worry about mobile
mobile doesn't have to deal with desktop

Smaller File sizes individually

desktop may be 400k
mobile may be 200k

Faster to code if new to mobile

responsive is tricky

Easier to tailor experience

like, "this report is unnecessary on mobile", remove it.

Easier to make decisions

don't have to consider all the things
or, "add a print button"?

Less to break when updating

testing is quicker

Allows for developers to "Specialize" in one base

better development when specializing

Responsive Code Base

Advantages

* good for sites that will likely be the same on both.
* experienced devs only
* a framework does not a responsive site make.

Requires good code organization

not fo yolo devs. keep css game tight.

Smaller file sizes combined

desktop may be 400k, mobile may be 200k,
but responsive would probably be 475k

More likely to net a better product in the end

I am a big believer in this
devs think better about design

One code base to maintain

don't get out of sync.
had issue where desktop would update, but not mobile, vice versa.

Generally accepted as the "recommended" method

google, basically.

Images

Where possible, use vectors!

SVG is Fully Suported

SVG > Icon Font (usually)

  • More Accessible
  • More Flexible
  • Multi-color
- More accessible (some users have forced fonts, so icons don't look weird if svg)
- more flexible (don't have to be in before or after, can be a background)
- BUT icon fonts are *slightly* better performance. & can change color easily.

1x, 2x, & 3x Images


  <img src="gob.jpeg"
       srcset="gob.jpeg 1x,
               gob@2x.jpeg 2x,
               gob@3x.jpeg 3x">
1x 2x 3x better supported than specific sizes.

Lazy Load


            <img data-src="gob.jpeg"
                 data-srcset="gob.jpeg 1x,
                         gob@2x.jpeg 2x,
                         gob@3x.jpeg 3x">

Allows for slick animations too!

they will all work about the same, img path goes in a data attr, and detects when to lad the images.
Bonus: can get slick animations if done well.

Transform

If possible, use transform.

Translate doesn't repaint.
hardware accelerated.

Not this


.c-menu {
  position: absolute;
  left: 0;
  transition: left .2s ease;
}
.c-menu.is-open{
  left: 50px;
}

This


.c-menu {
  position: absolute;
  left: 0;
  transition: transform .2s ease-out;
}
.c-menu.is-open{
  transform: translateX(50px);
}
